WEB 端适配 rem 方案 - 盛和塾数据大屏经验分享 ​

本次培训通过分享盛和塾数据大屏项目的适配改造经验，初步讲解了 rem 单位的基础知识和使用心得，提出了样式代码的规范意见。其中涉及到的知识点如下：

1.关于 rem 单位 ​

rem 是 css3 中新增加的一种相对单位，mdn 上对其描述为:

这个单位代表根元素（通常为<html> 元素）的 font-sze 大小。当用在根元素的 font-size 上面时 ，它代表了它的初始值。

html
<html>
  <body>
    <div id="app"></div>
  </body>
</html>

例如当前 html 标签的 font-size 为 10px，那么 1rem 就等于 10px。现在假设项目中所有标签元素的 css 属性单位都换成 rem，那么我们只需要控制 html 的 font-size 就可以实现对全部元素的等比例缩放，起到了四两拨千斤的效果。

那么如何控制 rem 在不同分辨率下的指呢？这里就需要使用媒体查询技术。

2.关于媒体查询在方案中的应用 ​

媒体查询的具体语法此处不再赘言，具体请参阅 MDN 上的说明

具体代码如下：

scss
// 此处使用 scss

$HTML_fontSize: 20px;

@media screen and (max-width: 1023px) {
  $scale: 0.5;

  html {
    font-size: $HTML_fontSize * $scale;
  }
}
@media screen and (min-width: 1024px) {
  $scale: 0.6;

  html {
    font-size: $HTML_fontSize * $scale;
  }
}
@media screen and (min-width: 1280px) {
  $scale: 0.7;

  html {
    font-size: $HTML_fontSize * $scale;
  }
}
@media screen and (min-width: 1440px) {
  $scale: 0.8;

  html {
    font-size: $HTML_fontSize * $scale;
  }
}
@media screen and (min-width: 1600px) {
  $scale: 0.9;

  html {
    font-size: $HTML_fontSize * $scale;
  }
}
@media screen and (min-width: 1920px) {
  $scale: 1;

  html {
    font-size: $HTML_fontSize * $scale;
  }

通常我们的 UI 设计师在没有明确要求的情况下出的 web 设计图是在 1920×1080 分辨率环境下。这样我们就以该分辨率作为基准，设置 html 的 fontsize 为 20px。

为什么用 20px？ 因为好算。例如：17px = 0.85rem

然后根据项目的分辨率要求（或者个人习惯），在不同的分辨率范围内调整基准 fontsize 的系数，这个过程的本质就是在做宽度的比例缩放。因此值得注意的是，考虑到屏幕比例（16:9，16:10，4:3）不同，这里贴出的步长 0.1 的递减关系属于经验值，并不是在所有项目或条件下都适用，请务必要根据实际情况进行判断。

如果您发现了更好的系数设置，欢迎分享宝贵经验。

将该 scss 文件引入到 main.js 文件中即可实现全局引入，根据实际情况调整原来使用 px 绝对单位的元素属性即完成了 rem 方案的适配。

css
// main.js
// 全局样式
import "@/styles/adaptive.scss";

在数据大屏项目适配的过程中，发现整个项目的 html 部分几乎都在用内联样式写 style。从个人经验出发，内联样式可以用，但所有样式都这么写它会有两个明显坏处：

  1. 在最大程度上拉满了项目的维护难度，导致后续开发变得困难重重，让开发人员生不如死 html 提供 class 属性的目的正如其名字一样，class，类，让样式相同的元素共享一个样式类，低耦合高聚合的开发原则的终极体现。
  2. 内联样式的权值极高，可能导致样式无法实现还找不到的问题 关于权值（优先级）请参考 mdn。简单来说在不考虑!impornt 的情况下，内联样式 1000 ＞ ID 选择器 100 ＞类选择器 10 ＞ HTML 标签选择器 1。导致 css 样式修改失败的三大原因之一就是用了内联样式而不自知。

因此，个人建议阅读此文的朋友们请养成写 class 的好习惯。

5.关于适配的其他说明 ​

本次项目适配除了 rem 和媒体查询，还使用了 grid 栅格布局。grid 布局的优点非常突出，使用 fr 自适应单位让适配工作如虎添翼。国内代码科普界一哥阮一峰对此种布局有详细的阐述，本文不再班门弄斧，请各位阅读学习他的专题教程《CSS Grid 网格布局教程》。
